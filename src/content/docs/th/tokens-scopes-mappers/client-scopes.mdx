## Client Scopes คืออะไร

Client scope คือชุดของ mapper ที่ตั้งชื่อและนำกลับมาใช้ซ้ำได้ ช่วยให้กำหนด claim ชุดหนึ่งครั้งแล้ว attach ให้ client หลายตัวได้ แทนที่จะกำหนด mapper ซ้ำในแต่ละ client

## Default vs Optional Scope

- **Default scope** จะรวมอยู่เสมอในทุก token request โดยไม่ต้องระบุเพิ่มเติม
- **Optional scope** จะรวมเฉพาะเมื่อ authorization request ระบุอย่างชัดเจนผ่าน scope parameter

## Built-in Scopes

| Scope | สิ่งที่เพิ่มลงใน Token |
|---|---|
| `profile` | เพิ่ม name, given_name, family_name, preferred_username |
| `email` | เพิ่ม email, email_verified |
| `roles` | เพิ่ม realm_access, resource_access |
| `openid` | จำเป็นสำหรับ OIDC |
| `offline_access` | ให้ refresh token ที่อยู่ได้นาน |

## วิธี Assign Optional Scope ให้ Client

1. ไปที่ **Clients** → เลือก client ของคุณ
2. คลิก tab **Client scopes**
3. คลิก **Add client scope**
4. เลือก scope ที่ต้องการแล้วตั้งเป็น **Optional**
5. คลิก **Add**

## วิธีสร้าง Custom Scope

1. ไปที่ **Client scopes** → คลิก **Create client scope**
2. ตั้งชื่อ scope เช่น `orders:read`
3. เลือก **Protocol**: `openid-connect`
4. คลิก **Save**
5. ไปที่ tab **Mappers** เพื่อเพิ่ม mapper ลงใน scope นี้
