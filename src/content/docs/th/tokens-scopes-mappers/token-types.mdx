## ประเภทของ Token ใน Keycloak

Keycloak ออก token สามประเภทในขั้นตอน OIDC flow แต่ละประเภทมีวัตถุประสงค์ต่างกัน

## Access Token

ใช้อนุญาต API call — เป็น JWT อายุสั้นที่ API ของคุณตรวจสอบในทุก request Bearer token นี้คือสิ่งที่คุณส่งไปใน `Authorization` header เมื่อเรียก backend service

## ID Token

บอก client ว่าผู้ใช้คือใคร — ใช้ได้เฉพาะใน client เท่านั้น เช่น เพื่อแสดงชื่อผู้ใช้ใน UI ห้ามส่ง ID token ไปยัง backend API เด็ดขาด

## Refresh Token

อายุยาวกว่า access token — ใช้ขอ access token ใหม่โดยไม่ต้อง login ซ้ำ ส่งได้เฉพาะไปยัง Keycloak token endpoint เท่านั้น

## ตัวอย่าง Decoded JWT

ด้านล่างนี้คือ access token ที่ถูก decode แล้ว แสดง header, payload และ claim ต่างๆ ที่ Keycloak ใส่ไว้:

## Claim สำคัญใน Access Token

| Claim | ความหมาย |
|---|---|
| `sub` | Unique ID ของผู้ใช้ |
| `iss` | URL ของ realm ที่ออก token |
| `aud` | ผู้รับ token ที่ตั้งใจ |
| `exp` | เวลาที่ token หมดอายุ (Unix timestamp) |
| `azp` | Client ที่ขอ token |
| `scope` | Scope ที่ถูก grant |
| `realm_access.roles` | Realm roles ของผู้ใช้ |

<Callout title="ส่ง access token ไปยัง API ไม่ใช่ ID token">
ส่ง access token ไปยัง API ของคุณเป็น Bearer token ใน Authorization header ใช้ ID token เฉพาะใน client เพื่อแสดงข้อมูลผู้ใช้ — อย่าส่ง ID token ไปยัง backend service เด็ดขาด
</Callout>
