## ภาพรวมโมดูล

ใน Keycloak ทุก request ที่ผ่าน API จะถูกตรวจสอบด้วย access token โมดูลนี้จะสอนว่า token ทำงานอย่างไร scope กำหนดว่า claim ใดจะถูกใส่เข้าไปใน token และ mapper เป็นเครื่องมือที่เพิ่มหรือแปลง claim เหล่านั้น

## สิ่งที่คุณจะเรียนรู้

- เข้าใจความแตกต่างระหว่าง access token, ID token และ refresh token
- กำหนด default และ optional client scope
- เพิ่ม protocol mapper เพื่อใส่ claim ลงใน token
- ตั้งค่า lifespan ของ token ให้เหมาะสม

<Callout title="Access token คือสิ่งที่ API ของคุณตรวจสอบ">
Access token คือ token ที่ API ตรวจสอบในทุก request — ออกแบบ claim ที่อยู่ใน token อย่างตั้งใจ ทุก claim ที่ปรากฏอยู่ใน token มาจาก scope หรือ mapper
</Callout>
