วิธีที่เร็วที่สุดในการรัน GrowthBook บนเครื่องของคุณคือใช้ Docker Compose สแตก Compose ด้านล่างนี้จะเริ่ม GrowthBook และ MongoDB ขึ้นมาพร้อมกัน — ไม่ต้องเชื่อมต่อ database เองเลย

## ขั้นตอนทีละขั้น

1. **บันทึก compose.yaml** ที่แสดงในกล่องด้านบนไว้ที่ root ของโปรเจกต์คุณ ไฟล์นี้นิยาม service สองตัว — `mongo` และ `growthbook` — และ named volume หนึ่งตัวสำหรับเก็บไฟล์ที่อัปโหลด

2. **เริ่ม GrowthBook แบบทำงานเบื้องหลัง** โดยรัน:

```bash
docker compose up -d
```

   flag `-d` (detached mode) จะคืนการควบคุมกลับมาที่ terminal ของคุณทันทีในขณะที่ทั้งสอง container เริ่มทำงานอยู่เบื้องหลัง

3. **เปิด browser ไปที่ `http://localhost:3000`** — นี่คือ GrowthBook web UI ที่คุณใช้สร้าง feature flags, รัน experiments และจัดการบัญชีของคุณ

4. **SDK ของคุณจะสื่อสารกับ `http://localhost:3100`** — นี่คือ endpoint ของ GrowthBook SDK API แอปของคุณจะดึง payload ของ feature flag จาก URL นี้ ไม่ใช่จาก port 3000

5. **หยุดและตรวจสอบสแตก** เมื่อคุณทำเสร็จแล้ว:

```bash
# Stop all containers (data is preserved in the uploads volume)
docker compose down

# Tail live logs for the growthbook container only
docker compose logs -f growthbook
```

## compose.yaml ฉบับเต็มสำหรับอ้างอิง

```yaml
services:
  mongo:
    image: mongo:latest
    environment:
      - MONGO_INITDB_ROOT_USERNAME=root
      - MONGO_INITDB_ROOT_PASSWORD=password
  growthbook:
    image: growthbook/growthbook:latest
    ports: ["3000:3000", "3100:3100"]
    depends_on: [mongo]
    environment:
      - MONGO_URL=mongodb://root:password@mongo:27017/
    volumes: ["uploads:/usr/local/src/app/packages/back-end/uploads"]
volumes:
  uploads:
```

## ตารางอ้างอิง Port

| Port | Service | จุดประสงค์ |
|------|---------|---------|
| 3000 | GrowthBook web app | Admin UI — สร้าง flags, experiments, metrics |
| 3100 | GrowthBook SDK API | แอปของคุณดึง feature flags จากที่นี่ |
| 27017 | MongoDB (ภายใน) | ไม่ได้ expose ออกสู่ host; ใช้โดย container ของ GrowthBook เท่านั้น |

## เกี่ยวกับ MongoDB

GrowthBook เก็บ configuration ของตัวเองทั้งหมดไว้ใน MongoDB — นิยามของ feature flag, config ของ experiment, metrics และการตั้งค่าของทีม MongoDB **ไม่ได้** เก็บ analytics event data หรือ user data warehouse ของคุณ raw events ของคุณจะยังคงอยู่ที่เดิม (Snowflake, BigQuery, Redshift ฯลฯ)
