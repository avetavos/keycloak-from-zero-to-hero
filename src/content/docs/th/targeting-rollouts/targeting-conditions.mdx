## Targeting Conditions

targeting condition คือตัวกรองที่ตัดสินว่า rule จะใช้กับผู้ใช้คนไหน แต่ละ rule บน Feature สามารถมี condition ได้ตั้งแต่ศูนย์ตัวขึ้นไป **ทุก condition บน rule ต้องเป็นจริง** rule นั้นจึงจะ match กับผู้ใช้คนปัจจุบัน หาก condition ใดเป็นเท็จ ผู้ใช้คนนั้นจะไม่ match rule นั้นและ GrowthBook จะไปทำ rule ถัดไป

### การเพิ่ม condition ใน GrowthBook UI

1. ในหน้ารายละเอียดของ Feature คลิก **Add Rule** (หรือคลิกไอคอนรูปดินสอบน rule ที่มีอยู่)
2. ในตัวแก้ไข rule คลิก **Add Condition**
3. เลือก **Attribute** จาก dropdown (เช่น `country`)
4. เลือก **Operator** (equals, not equals, contains, regex และอื่น ๆ)
5. ใส่ **Value** (เช่น `TH`)
6. คลิก **Save Rule**

คุณสามารถเพิ่ม condition ได้มากเท่าที่ต้องการ แต่ละ condition ที่เพิ่มเข้ามาจะจำกัด audience ให้แคบลงไปอีก

### Condition operators

GrowthBook รองรับ operator หลากหลายเพื่อ match attribute หลายประเภท:

- **equals** / **does not equal** — match แบบตรงทั้งหมดบน string, number, หรือ boolean
- **contains** / **does not contain** — match substring สำหรับ string
- **matches regex** — ทดสอบ string กับ regular expression
- **is greater than** / **is less than** — การเปรียบเทียบเชิงตัวเลข
- **is in list** / **is not in list** — ตรวจสอบว่าค่าปรากฏอยู่ในชุดที่คั่นด้วยจุลภาคหรือไม่

### Advanced JSON mode

สำหรับ power user GrowthBook รองรับ condition JSON แบบ MongoDB-style ซึ่งเป็นรูปแบบเดียวกับที่ engine การประเมินใช้ภายใน คลิก **Advanced** ในตัวแก้ไข rule เพื่อป้อน JSON ดิบแทนการใช้ visual builder

match ผู้ใช้ที่ `country` เป็น `TH` **และ** `plan` เป็น `pro`:

```json
{
  "country": "TH",
  "plan": "pro"
}
```

match ผู้ใช้ที่ `email` ลงท้ายด้วย `@company.com`:

```json
{
  "email": { "$regex": "@company\\.com$" }
}
```

ตัวแก้ไขแบบ advanced มีประโยชน์สำหรับนิพจน์ที่ซับซ้อน (เช่น `$or`, `$and`, `$not` แบบซ้อนกัน) ที่ visual builder ไม่ได้เปิดให้ใช้
