## Percentage Rollouts

rule แบบ percentage rollout ให้คุณปล่อย flag ให้ผู้ใช้เพียงบางส่วน แทนที่จะเปิด flag ให้ทุกคนพร้อมกัน คุณสามารถเริ่มที่ 10% ตรวจสอบว่าระบบยังทำงานปกติดี แล้วค่อยขยับเป็น 50% และสุดท้าย 100%

### การสร้าง rollout rule ใน GrowthBook

1. ในหน้ารายละเอียดของ Feature คลิก **Add Rule**
2. เลือก **Rollout (percentage)** เป็นประเภทของ rule
3. ตั้งค่าเปอร์เซ็นต์ **Coverage** (เช่น 10)
4. ตั้งค่า **Hash Attribute** — นี่คือ attribute ของผู้ใช้ที่จะนำค่าไป hash เพื่อตัดสินการกำหนด bucket (โดยปกติคือ `id`)
5. เพิ่ม **Targeting Conditions** เพื่อจำกัด rollout ให้กับผู้ใช้บางกลุ่มได้ตามต้องการ
6. คลิก **Save Rule**

### การแบ่ง bucket ทำให้ผู้ใช้ได้ผลลัพธ์ที่สม่ำเสมออย่างไร

GrowthBook จะ hash ค่าของ hash attribute ของผู้ใช้ (เช่น `id` ของพวกเขา) รวมกับ feature key เพื่อสร้างตัวเลขระหว่าง 0 ถึง 1 หากตัวเลขนั้นต่ำกว่าเปอร์เซ็นต์ coverage rule จะ match เนื่องจาก hash เป็นแบบ deterministic ผู้ใช้คนเดิมจะตกลงใน bucket เดิมเสมอ การเพิ่ม coverage จาก 10% เป็น 50% จะเพิ่มเฉพาะผู้ใช้ใหม่เท่านั้น ไม่เคยพลิกสถานะของผู้ใช้เดิม

### การ ramp rollout

ตารางการ ramp ทั่วไป: เริ่มที่ 10% เป็นเวลา 24 ชั่วโมง คอยดู error rate และ metric หากปกติดี ขยับเป็น 50% อีก 24 ชั่วโมง จากนั้น rollout ไปที่ 100% และลบ flag ออก

```js
gb.setAttributes({ id: 'user-123' });
if (gb.isOn('new-dashboard')) {
  // user-123 is consistently in the 10% bucket
}
```
